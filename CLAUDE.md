# portfolio-dashboard

Статический сайт (без сборки): `index.html`, `app.js`, `config.js`, `style.css`.
Хостинг: GitHub Pages из ветки `main` — https://vladslastyon-jpg.github.io/portfolio-dashboard/

## Рабочий процесс

1. Пользователь описывает задачу (может быть как точное ТЗ, так и общая цель).
2. Правки вносятся в нужные файлы.
3. После завершения правок — увеличить номер `?v=` у изменённых файлов в тех местах,
   где они подключаются в `index.html` (`style.css?v=N`, `config.js?v=N`, `app.js?v=N`),
   чтобы обойти кэш браузера/CDN.
4. Коммит с понятным сообщением и пуш в `origin/main` — без дополнительного
   подтверждения на каждый пуш (стоящее согласие пользователя на этот процесс).
5. GitHub Pages пересобирает сайт из `main` автоматически (обычно до минуты).
